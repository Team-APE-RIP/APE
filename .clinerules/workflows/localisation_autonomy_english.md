# 进行HOI4 MOD的国家名称英文本地化翻译补全

## 规则

0. 所有代码必须符合HOI4 Modding语法。所有修改限制在工作区的mod内部。翻译时只可以修改翻译出来的文件，不可修改原件（要翻译的文件）。
1. 对应语言的文件为`_l_<LANG>.yml`结尾和格式，必须严格使用UTF-8-BOM编码。
2. 本地化文件顶部要有`l_<LANG>:`。
3. 后续所有本地化每个开头要空一行。
4. `$<KEY>$`是引用本地化键的意思，这时候读取对应的本地化键，根据对应语言正确语序判断它应该在哪里。对于完全相同的文本可以巧妙使用`$<KEY>$`进行替换，对于simp_chinese中使用了`$<KEY>$`但在翻译语言中不能完美符合的可以不用。
5. 如果`$<KEY>$`中的内容来自mod内其他的文件夹，一并翻译，文件处理参考步骤4，并且必须在对话中特别强调！
6. 所有的引号（“和”）在英文中应该使用`\"`替换。

## 提示

0. 所有翻译应该言简意赅，因为只是一个名字而不是一句话。
1. 所有翻译应该符合原本含义，做到信达雅。
2. 这是一个处于2050年的悲惨的世界的世界观的作品，如果是现实世界中在2025年存在的体制，翻译应该保持不变；不存在的或历史上存在过的，你可以自行思考如何翻译。
3. 对于世界观问题，你可以在我的博客的Mod介绍页面"https://czxieddan.top/APE/"中浏览。
4. 对于代码问题，你可以在Paradox的Wiki网站的Modding中搜索。

## 交互

0. 如果文件过多，可以一次先翻译其中一部分。但是不要遗漏。

1. 如果在翻译过程中有疑问，你可以停下来问我，或者把有问题的部分留到最后问我。

2. 批量将需要修改/新建的文件使用翻译文件的powershell脚本：(示例）
   
   ```
   # TAG - NAME
   $content = @"
   l_english:
   
    KEY: "TEXT"
    KEY2: 'TEXT2"
   "@
   [System.IO.File]::WriteAllText($dir + "TAG_l_english.yml", $content, $utf8bom)
   ···
   ```
   
    避免无法创建UTF-8-BOM编码文件和新建文件过多占用token。
   
    运行完脚本后记得把临时脚本文件删除。
3. 压缩对话时不要遗忘工作流，一定要声明这个工作流。
   

## 步骤

1. 读取`localisation/simp_chinese/common/autonomy/`里的所有文件。
2. 读取`localisation/english/common/autonomy/`里的所有文件。
3. 判断`localisation/simp_chinese/common/autonomy/`里的所有文件的本地化键（KEY）是否有出现在`localisation/english/common/autonomy/`里的本地化文件里（注意不要只检查文件名字，还要看里面的内容，里面的每个KEY是否都有）。
4. 如果有缺少的，在同名文件里补充或创建同名文件（位置在对应的语言文件夹内，相对路径相同）补充其对应的english翻译。（注意，此处的同名文件指的是`_l_<LANG>`的语言声明之前的部分。）；如果有文件不是UTF-8-BOM编码，比如是UTF-8的，替换为UTF-8-BOM编码。
5. 在对话框中告诉我你对哪个文本进行了翻译，你的思路是什么。
